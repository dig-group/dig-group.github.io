# 更新日志目录说明

这里用于记录 DIG 官网后续每次更新，建议每次变更新增一条独立日志文件，文件名使用日期 + 简短主题。

## 记录规范

每条日志建议包含以下字段：

- 修改时间
- 修改目的
- 修改内容
- 修改结果
- 相关文件
- 验证方式

## 文件命名建议

```text
YYYY-MM-DD-主题.md
```

示例：

```text
2026-05-09-showcase-redesign.md
```

## 维护要求

后续如果有新的内容更新、图片替换、样式调整或交互优化，都应同步在 `updateLog/` 下追加一条日志，便于后续维护人员追踪变更历史。
